## 维护规则

1. SOP 写项目真实用到的节点、脚本和参数。
2. 已实装系统写“当前落点”和“验收标准”。
3. 未实装系统写“预备模板”，等完成后补真实路径。
4. 每次系统升级后，同步更新对应 SOP，而不是另开散乱笔记。
5. 能复用的参数、节点结构、坑点都要写下来。
