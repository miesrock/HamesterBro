# UI 菜单模板 SOP

## 适用场景

用于 Survivors 原型中的战斗 HUD、Debug 面板、背包格、升级菜单和暂停菜单。

## 当前项目落点

- 脚本：`hamsterbroproject/scripts/debug_ui.gd`
- HUD：`hamsterbroproject/scripts/ui_hud.gd`
- 升级菜单：`hamsterbroproject/scripts/upgrade_menu.gd`
- 暂停菜单：`hamsterbroproject/scripts/pause_menu.gd`
- 设置菜单：`hamsterbroproject/scripts/settings_menu.gd`
- 主场景节点：`DebugUI`

## 当前已完成

1. 左上 Health 面板，已接玩家 `health_changed` 信号。
2. 左上 Synergy Active 面板。
3. 简化背包格。
4. 右上 FPS / Debug / Juice 文本。
5. 左下 Debug 面板。
6. 中部规则大字，模拟参考图压屏感。
7. 左上经验条，已接玩家 `experience_changed` 和 `level_up` 信号。
8. 升级三选一菜单，升级时暂停弹出。
9. Esc 暂停菜单。
10. Settings 菜单可切换 debug overlay 显示。

## 当前定位

当前 `DebugUI` 是 UI 根协调器，具体面板已经拆到 `HUD` / `UpgradeMenu` / `PauseMenu` / `SettingsMenu`。它的价值是快速验证：

- 战斗时文字是否可读。
- UI 是否遮挡主角。
- 屏幕信息密度是否接近参考图。
- FPS 和敌人数量是否能实时观察。
- 经验球收集后经验条和等级文本是否实时变化。

## 当前拆分

```text
UI CanvasLayer
├─ HUD
│  ├─ HealthPanel
│  ├─ ExperienceBar
│  ├─ InventoryGrid
│  └─ DebugPanel
├─ UpgradeMenu
├─ PauseMenu
└─ SettingsMenu
```

## 实现步骤

1. `CanvasLayer` 作为 UI 根节点。
2. 战斗常驻信息放 `HUD`。
3. 升级选择放 `UpgradeMenu`，默认隐藏。
4. 暂停和设置放独立菜单，默认隐藏。
5. UI 通过 signal 接收玩家经验、等级、生命值变化。
6. Debug 文本保留导出开关，方便录宣传素材时隐藏。
7. 升级菜单通过 `upgrade_selected(upgrade_id)` 通知升级系统，不直接改玩家或战斗参数。

## 验收标准

- 1280x720 下 UI 不遮挡主角。
- 字体有黑色描边，在浅色训练场上可读。
- 伤害数字和 HUD 尺寸有层级差异。
- 左上信息、右上性能、左下 debug 三个区域互不抢。
- 血量条显示 `HP 当前 / 最大`。
- 经验条显示 `LV 等级 XP 当前 / 需求`。
- UI 通过 signal 刷新，不直接写玩家状态。
- Esc 可以打开/关闭暂停菜单。
- 升级菜单出现时暂停菜单不能抢输入。

## 可扩展方向

- 将 Settings 菜单接入持久化设置。
- 增加音量、屏幕震动、伤害数字密度、敌人数量上限。
- 为升级选项加图标、稀有度和随机池。
