# 玩家移动手感模板 SOP

## 适用场景

适用于 3D 斜俯视 Survivors / ARPG / 小型战斗原型。目标是先获得可靠的移动、跳跃、朝向和模型调试结构，而不是做复杂动画状态机。

## 当前项目落点

- 场景：`hamsterbroproject/scenes/Player.tscn`
- 脚本：`hamsterbroproject/scenes/player.gd`
- 输入：`move_forward`、`move_back`、`move_left`、`move_right`、`jump`
- 角色组：`player`

## 推荐节点结构

```text
Player CharacterBody3D
├─ CollisionShape3D
├─ MeshInstance3D              # 占位体，导入模型后隐藏
├─ CameraPivot
│  └─ Camera3D
└─ VisualPivot                 # 只负责根据移动方向旋转
   ├─ HamsterModel             # 美术模型，允许单独调位置/比例
   └─ Muzzle                   # 枪口/技能出生点，允许单独调位置
```

## 核心参数

```gdscript
@export var move_speed: float = 4.0
@export var jump_velocity: float = 7.0
@export var gravity: float = 22.0
@export var visual_yaw_offset_degrees: float = 180.0
```

## 实现步骤

1. Player 使用 `CharacterBody3D`。
2. 在 `_ready()` 中 `add_to_group("player")`，方便经验球、敌人、生成器查找。
3. WASD 只控制 XZ 平面速度。
4. Space 映射到 `jump`，接地时设置 `velocity.y = jump_velocity`。
5. 没有真实地板碰撞时，用初始 `_ground_y` 做临时地面高度。
6. 角色朝向只旋转 `VisualPivot`，不要直接旋转导入的 `HamsterModel`。
7. 如果模型正面和移动方向相反，调 `visual_yaw_offset_degrees`，常见值是 `0` 或 `180`。
8. 枪口位置单独放在 `VisualPivot/Muzzle`，在编辑器里直接拖节点调试。

## 验收标准

- WASD 可平滑移动。
- Space 可跳起并落回地面。
- 模型朝向跟随移动方向。
- 调整 `HamsterModel` 的本地位置不会破坏移动朝向。
- 调整 `Muzzle` 的本地位置会改变子弹出生点。

## 常见坑

- 导入模型自带偏移：不要用脚本每帧改 `HamsterModel.position`，只旋转父节点 `VisualPivot`。
- 模型正反相反：优先改 `visual_yaw_offset_degrees`，不要来回改 GLB 根节点。
- 地面没有碰撞：短期可用 `_ground_y`，长期应给训练场地面加 `StaticBody3D`。
