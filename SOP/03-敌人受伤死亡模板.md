# 敌人受伤/死亡模板 SOP

## 适用场景

用于 Survivors 类游戏的轻量敌人：数量多、AI 简单、死亡频繁、反馈要清楚。

## 当前项目落点

- 敌人脚本：`hamsterbroproject/scripts/enemy.gd`
- 子弹命中：`hamsterbroproject/scripts/bullet.gd`
- 掉落触发：`hamsterbroproject/scripts/combat_director.gd`

## 推荐节点结构

当前敌人是运行时纯脚本生成，适合原型：

```text
Enemy CharacterBody3D
├─ MeshInstance3D outline
├─ MeshInstance3D body
└─ MeshInstance3D shell/detail
```

量产后可改成 `.tscn`：

```text
Enemy CharacterBody3D
├─ CollisionShape3D
├─ VisualPivot
│  └─ Model
└─ Hurtbox Area3D
```

## 核心逻辑

1. `_ready()` 中 `add_to_group("enemies")`。
2. `_physics_process()` 中朝玩家移动。
3. 子弹通过 `"enemies"` 组查找目标。
4. 命中后调用：

```gdscript
func take_hit(hit_direction: Vector3) -> void:
	_knockback = hit_direction.normalized() * 4.5
	queue_free()
```

## 推荐扩展顺序

1. 当前阶段：命中立即死亡。
2. 下一阶段：加入 `max_health` / `current_health`。
3. 再下一阶段：死亡前播放 0.12-0.2 秒缩放弹出动画。
4. 再下一阶段：通过 `died(global_position)` signal 交给掉落系统，而不是由 `CombatDirector` 直接生成掉落。

## 验收标准

- 100+ 敌人时依然流畅。
- 子弹命中后敌人能稳定消失。
- 敌人死亡位置能正确生成伤害字、火花和经验球。
- 敌人材质和训练场有区分，轮廓在浅色地面上可读。

## 常见坑

- 敌人一死就 `queue_free()`，后续系统不能再读它的位置：命中回调里要先保存 `hit_position`。
- 敌人数量大时不要给每个敌人加复杂粒子和 UI。
- 组名统一使用 `"enemies"`，不要混用 `"enemy"`。
