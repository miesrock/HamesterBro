# 伤害数字/受击反馈模板 SOP

## 适用场景

用于快速提升战斗爽感：命中时出现描边文字、颜色变化、火花、音效、轻微屏幕信息压力。

## 当前项目落点

- 伤害数字：`hamsterbroproject/scripts/damage_text.gd`
- 命中反馈：`hamsterbroproject/scripts/combat_director.gd`
- 音效：`hamsterbroproject/scripts/juicy_audio.gd`

## 当前已完成

1. `DamageText` 继承 `Label3D`。
2. 开启 billboard 和 fixed size。
3. 使用黑色描边。
4. 向上漂浮并淡出。
5. 命中时随机显示：
   - `999!`
   - `777!`
   - `1337!`
   - `OUCH!`
   - `CRIT-HAM!`
6. 命中生成 3 个彩色火花。
7. 根据普通命中/暴击播放不同音效。

## 推荐参数

```gdscript
@export var float_speed: float = 2.4
@export var lifetime: float = 0.7
font_size = 26
outline_size = 6
```

## 实现步骤

1. 子弹命中敌人后发出 `enemy_hit(enemy, hit_position)`。
2. `CombatDirector` 节流生成伤害字，避免满屏文字遮挡。
3. 根据文字类型设置颜色：
   - 普通：黄/橙。
   - `OUCH!`：白。
   - `CRIT-HAM!`：红橙。
4. 火花使用小 `BoxMesh`，短 tween 后缩放到 0 并释放。
5. 音效与视觉同时触发，形成命中闭环。

## 验收标准

- 命中后能立即看到文字反馈。
- 文字不会大到遮住主角和敌群。
- 多次命中时有节奏感，不是 UI 噪声墙。
- 火花位置跟命中点一致。
- 音效不会因为高频攻击而爆音。

## 常见坑

- 首版伤害字很容易太大：优先调 `font_size`、`outline_size`、`damage_text_interval`。
- `Label3D.fixed_size = true` 后屏幕占比稳定，但远近关系会被弱化，适合街机反馈。
- 大量文字应做对象池或上限控制。
