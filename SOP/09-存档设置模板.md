# 存档/设置模板 SOP

## 适用场景

用于保存玩家永久进度和本地设置。今日尚未实装，本 SOP 是后续落地模板。

## 推荐文件

```text
hamsterbroproject/scripts/save_service.gd
hamsterbroproject/scripts/settings_service.gd
```

推荐作为 Autoload：

```text
SaveService
SettingsService
```

## 存档内容

适合放在 `user://save.json`：

```json
{
  "version": 1,
  "total_runs": 0,
  "best_survival_time": 0.0,
  "unlocked_weapons": ["cheese_gun"],
  "meta_currency": 0
}
```

## 设置内容

适合放在 `user://settings.cfg`：

```text
[audio]
master_volume=0.8
sfx_volume=0.9

[video]
fullscreen=false
damage_text_density=1.0
screen_shake=0.7

[gameplay]
enemy_target_count=120
```

## 实现步骤

1. 新建 `SaveService.gd` Autoload。
2. `_ready()` 时读取 `user://save.json`。
3. 如果文件不存在，创建默认数据。
4. 每次局外进度变化后调用 `save_game()`。
5. 新建 `SettingsService.gd` Autoload。
6. 设置变更后立刻应用并写入 `user://settings.cfg`。
7. UI 菜单只调用服务，不直接读写文件。

## 验收标准

- 第一次启动没有存档也不报错。
- 设置调整后重启仍然生效。
- 存档版本号存在，方便未来迁移。
- 损坏存档不会导致游戏启动失败，应回退默认值并输出 warning。

## 常见坑

- 不要把运行中临时数据全部写进永久存档。
- 不要每帧保存。
- 设置和存档分开，设置属于玩家偏好，存档属于游戏进度。
