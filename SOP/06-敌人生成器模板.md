# 敌人生成器模板 SOP

## 适用场景

用于 Survivors 类无限补怪：敌人始终围绕玩家在屏幕外或远处生成，形成持续压力。

## 当前项目落点

- 脚本：`hamsterbroproject/scripts/enemy_spawner.gd`
- 主场景节点：`EnemySpawner`

## 当前参数

```gdscript
@export_range(20, 300, 1) var target_count: int = 120
@export var spawn_min_radius: float = 18.0
@export var spawn_max_radius: float = 28.0
@export var refill_per_tick: int = 12
```

## 实现步骤

1. `EnemySpawner` 持有 `player_path`。
2. 每帧统计 `"enemies"` 组数量。
3. 如果低于 `target_count`，按 `refill_per_tick` 补齐。
4. 生成位置：
   - 随机角度 `0..TAU`。
   - 随机半径 `spawn_min_radius..spawn_max_radius`。
   - 位置 = 玩家位置 + 圆环偏移。
5. 给新敌人设置 `target = player`。

## 验收标准

- 玩家移动时，敌人会持续从玩家周围补充。
- 敌人不会直接刷在玩家脚下。
- 敌人数量能稳定维持在目标值附近。
- `target_count = 120` 时保持流畅。

## 性能建议

- 如果帧率明显下降，先降 `target_count` 到 80-100。
- 将 `refill_per_tick` 控制在 6-12，避免瞬间刷太多。
- 下一步可用计时器每 0.1 秒补怪，而不是每物理帧统计。
- 更大规模时改用对象池和空间分区。

## 可扩展方向

- 按时间增加 `target_count`。
- 按象限/屏幕边缘生成敌人。
- 加入精英怪生成权重。
- 生成器发出 `enemy_spawned(enemy)` signal，方便统计和特效。
