# 测试场景模板 SOP

## 适用场景

用于快速验证移动、攻击、敌群密度、UI 可读性和特效层次。风格参考街霸训练场：浅色地面、红蓝轴线、规则网格、黑白测试块、墙面 debug 信息。

## 当前项目落点

- 场景：`hamsterbroproject/scenes/test_environment.tscn`
- 脚本：`hamsterbroproject/scripts/test_environment.gd`
- 网格材质：`test_environment.tscn` 内的 `GridShader` / `GridMaterial`

## 视觉构成

1. 地面
   - 浅暖灰/米灰底色。
   - 1 米小格，5 米大格。
   - X/Z 轴线使用红蓝强识别色。

2. 测试块
   - 使用黑、白、灰三类 block。
   - 避免纯黑纯白，推荐：
     - 白：`#E6E4DC`
     - 黑：`#1E2026`
     - 灰：`#B8BAB4`

3. 墙面
   - 后墙/侧墙加强空间层次。
   - 加浅色 `Label3D` 测试文字。
   - 加青色半透明全息线框/图表装饰。

4. 环境光
   - 背景避免棕色。
   - 推荐浅灰蓝背景：`#C7CDD2`。
   - Ambient Light 可偏暖：`#D8D6CC`。

## 网格纹理逻辑和结构

当前训练场网格不是图片贴图，而是程序化 shader。好处是地面和墙面可以无限清晰缩放，不需要准备 texture，也不会因为 UV 拉伸导致格子变形。

### 资源结构

网格写在 `hamsterbroproject/scenes/test_environment.tscn` 里：

```text
TestEnvironment
├─ sub_resource Shader Sh_grid        # GridShader，负责算网格颜色
├─ sub_resource ShaderMaterial Mat_grid
├─ Floor MeshInstance3D               # material_override = Mat_grid
├─ BackWall MeshInstance3D            # material_override = Mat_grid
└─ LeftWall / RightWall MeshInstance3D # material_override = Mat_grid
```

如果以后要复用到多个场景，建议把 shader 拆成独立文件：

```text
hamsterbroproject/shaders/training_grid.gdshader
hamsterbroproject/materials/training_grid.tres
```

### 核心参数

```glsl
uniform vec4 bg_color = vec4(0.76, 0.75, 0.70, 1.0);
uniform vec4 grid_color = vec4(0.56, 0.59, 0.62, 1.0);
uniform vec4 major_grid_color = vec4(0.40, 0.45, 0.52, 1.0);
uniform vec4 x_axis_color = vec4(0.72, 0.18, 0.16, 1.0);
uniform vec4 z_axis_color = vec4(0.12, 0.32, 0.72, 1.0);

uniform float grid_size = 1.0;        # 小格 1 米
uniform float major_grid_size = 5.0;  # 大格 5 米
uniform float line_width = 0.018;
uniform float major_line_width = 0.035;
uniform float axis_width = 0.055;

uniform float fade_start = 60.0;
uniform float fade_end = 140.0;
```

### 坐标来源

shader 在 `vertex()` 里把模型顶点转成世界坐标：

```glsl
world_pos = (MODEL_MATRIX * vec4(VERTEX, 1.0)).xyz;
world_normal = normalize((MODEL_MATRIX * vec4(NORMAL, 0.0)).xyz);
```

这样网格跟随世界坐标，而不是跟随单个 mesh 的 UV。结果是：

- 地面、墙面、不同大小的 mesh 可以共享同一套格子比例。
- 移动或缩放 Plane/Box 时，格子仍按世界米数对齐。
- 红蓝轴线永远落在世界 X/Z 轴附近，方便 debug。

### 地面和墙面的投影选择

在 `fragment()` 里先默认使用地面坐标：

```glsl
vec2 p = world_pos.xz;
```

然后根据法线判断当前面朝向，如果是墙面就换成对应平面：

```glsl
vec3 n = abs(normalize(world_normal));

if (n.x > n.y && n.x > n.z) {
	p = world_pos.zy; # 左右墙：用 Z/Y 画网格
} else if (n.z > n.y) {
	p = world_pos.xy; # 前后墙：用 X/Y 画网格
}
```

这个逻辑很重要：如果墙面也用 `world_pos.xz`，墙上的网格会被压成一条线或方向错误。训练场要像“真实测试房间”，墙面必须按自己的立面坐标画格子。

### 网格线生成

小格和大格都通过 `fract()` 做周期线：

```glsl
float grid_line(vec2 pos, float size, float width) {
	vec2 coord = pos / size;
	vec2 derivative = fwidth(coord);
	vec2 grid = abs(fract(coord - 0.5) - 0.5) / derivative;
	float line = min(grid.x, grid.y);
	return 1.0 - smoothstep(width, width + 1.0, line);
}
```

逻辑拆解：

1. `pos / size` 把世界坐标转换成格子坐标。
2. `fract()` 取每个格子内部的重复位置。
3. `min(grid.x, grid.y)` 得到横线或竖线的最近距离。
4. `smoothstep()` 把线边缘做柔和过渡。
5. `fwidth()` 用屏幕空间导数做抗锯齿，远处网格不会疯狂闪。

### 大格、小格、轴线叠色顺序

颜色按层叠加：

```glsl
vec3 col = bg_color.rgb;
col = mix(col, grid_color.rgb, minor);
col = mix(col, major_grid_color.rgb, major);
col = mix(col, x_axis_color.rgb, x_axis);
col = mix(col, z_axis_color.rgb, z_axis);
```

顺序原则：

1. 底色最先。
2. 小格线第二层，颜色最轻。
3. 大格线第三层，颜色更深。
4. 红蓝轴线最后覆盖，保证 debug 识别度。

当前轴线逻辑：

```glsl
float x_axis = axis_line(world_pos.z, axis_width); # Z=0 的红线，表示 X 轴方向
float z_axis = axis_line(world_pos.x, axis_width); # X=0 的蓝线，表示 Z 轴方向
```

### 远处淡化

为了避免远处网格太密，最后按相机距离把颜色混回底色：

```glsl
float dist = distance(CAMERA_POSITION_WORLD, world_pos);
float fade = 1.0 - smoothstep(fade_start, fade_end, dist);
col = mix(bg_color.rgb, col, fade);
```

推荐参数：

```text
fade_start = 60.0
fade_end = 140.0
```

### 调参优先级

如果画面太吵：

1. 先减小 `line_width`。
2. 再降低 `grid_color` 对比度。
3. 再提高 `fade_start` / 降低 `fade_end` 控制远处淡化。
4. 最后才改 `grid_size`，因为格子尺寸会影响玩家移动和场景比例感。

如果主角不突出：

1. 让 `bg_color` 稍暖。
2. 降低小格线对比。
3. 保留红蓝轴线，但把 `axis_width` 控制在 `0.05-0.06`。
4. 避免墙面文字和地面网格同时高对比。

### 可复用 shader 骨架

```glsl
shader_type spatial;
render_mode cull_back, diffuse_lambert, specular_disabled;

uniform vec4 bg_color : source_color = vec4(0.76, 0.75, 0.70, 1.0);
uniform vec4 grid_color : source_color = vec4(0.56, 0.59, 0.62, 1.0);
uniform vec4 major_grid_color : source_color = vec4(0.40, 0.45, 0.52, 1.0);
uniform vec4 x_axis_color : source_color = vec4(0.72, 0.18, 0.16, 1.0);
uniform vec4 z_axis_color : source_color = vec4(0.12, 0.32, 0.72, 1.0);

uniform float grid_size = 1.0;
uniform float major_grid_size = 5.0;
uniform float line_width = 0.018;
uniform float major_line_width = 0.035;
uniform float axis_width = 0.055;
uniform float fade_start = 60.0;
uniform float fade_end = 140.0;

varying vec3 world_pos;
varying vec3 world_normal;

void vertex() {
	world_pos = (MODEL_MATRIX * vec4(VERTEX, 1.0)).xyz;
	world_normal = normalize((MODEL_MATRIX * vec4(NORMAL, 0.0)).xyz);
}

float grid_line(vec2 pos, float size, float width) {
	vec2 coord = pos / size;
	vec2 derivative = fwidth(coord);
	vec2 grid = abs(fract(coord - 0.5) - 0.5) / derivative;
	float line = min(grid.x, grid.y);
	return 1.0 - smoothstep(width, width + 1.0, line);
}

float axis_line(float value, float width) {
	float d = abs(value);
	float aa = fwidth(value);
	return 1.0 - smoothstep(width, width + aa, d);
}

void fragment() {
	vec3 n = abs(normalize(world_normal));
	vec2 p = world_pos.xz;

	if (n.x > n.y && n.x > n.z) {
		p = world_pos.zy;
	} else if (n.z > n.y) {
		p = world_pos.xy;
	}

	float minor = grid_line(p, grid_size, line_width);
	float major = grid_line(p, major_grid_size, major_line_width);
	float x_axis = axis_line(world_pos.z, axis_width);
	float z_axis = axis_line(world_pos.x, axis_width);

	vec3 col = bg_color.rgb;
	col = mix(col, grid_color.rgb, minor);
	col = mix(col, major_grid_color.rgb, major);
	col = mix(col, x_axis_color.rgb, x_axis);
	col = mix(col, z_axis_color.rgb, z_axis);

	float dist = distance(CAMERA_POSITION_WORLD, world_pos);
	float fade = 1.0 - smoothstep(fade_start, fade_end, dist);
	col = mix(bg_color.rgb, col, fade);

	ALBEDO = col;
	ROUGHNESS = 1.0;
}
```

## 实现步骤

1. 用一个独立 `TestEnvironment` 场景封装训练场。
2. 地面和墙体尽量作为可替换模块，不和玩家/敌人逻辑耦合。
3. 用 `ShaderMaterial` 做程序化训练场网格，不依赖图片贴图。
4. 地面和墙体复用同一个 `GridMaterial`，shader 内用法线判断投影平面。
5. 在 `_ready()` 中生成墙面 debug art。
6. 墙面文字使用 `Label3D`，颜色透明度控制在 0.5-0.7，避免抢主体。
7. 全息装饰使用 `StandardMaterial3D`，开启 `emission` 和 alpha transparency。

## 验收标准

- 斜俯视相机下能清楚看出网格和轴线。
- 敌群和子弹出现后，地面网格不会抢戏。
- 墙面网格方向正确，不会被压成横线或竖线。
- 远处网格有淡化，不产生明显噪点和闪烁。
- 主角暖色模型能从灰色训练场里跳出来。
- 远处墙面有信息密度，但不影响战斗阅读。

## 可复用原则

- 测试场景是“玩法显微镜”，不是最终关卡。
- 所有新增玩法系统都优先在该场景验收：移动、碰撞、弹幕、UI、性能、截图。
