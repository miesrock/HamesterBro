# 今日开发总结 - Vampire Survivors 视觉战斗切片

日期：2026-05-21

## 今日目标

把现有 Godot 4.3 项目从“街霸测试场原型”推进到“看起来已经像 Vampire Survivors 战斗”的视觉切片。重点不是完整数值系统，而是先把一屏内的观感做出来：仓鼠主角、敌群、自动彩色攻击、伤害数字、经验球吸附、Debug UI、训练场墙面信息和 Juicy 音效。

## 已完成系统

1. 玩家移动手感模板
   - WASD 平面移动。
   - Space 跳跃。
   - 玩家加入 `player` 组。
   - `VisualPivot` 负责跟随移动方向旋转，`HamsterModel` 可以继续单独调位置、比例和朝向。
   - `Muzzle` 独立成玩家子节点，用来调枪口/弹幕出生点。

2. 测试场景模板
   - 保留街霸训练场方向：浅色地面、红蓝轴线、规则网格、测试块。
   - 墙面加入测试文字和青色全息调试图形。
   - 当前作为视觉验证场景，不承担复杂碰撞关卡逻辑。

3. 敌人受伤/死亡模板
   - 敌人是轻量运行时生成 `CharacterBody3D`。
   - 敌人加入 `enemies` 组。
   - `take_hit(hit_direction)` 接收命中方向，触发击退意图并销毁。
   - 目前死亡立即 `queue_free()`，之后可扩展死亡动画、掉落表、血量。

4. 经验球/升级模板
   - 敌人被子弹命中后生成 `ExperienceOrb`。
   - 经验球会在距离玩家一定范围内自动吸附。
   - 收集后调用玩家 `add_experience(value)`。
   - 当前只累计经验值，升级 UI 和升级选择面板还未接入。

5. 自动武器模板
   - `CombatDirector` 每隔固定时间寻找最近敌人。
   - 按 `burst_count` 发射彩色喷射弹。
   - 子弹颜色循环 cyan / magenta / yellow / green / orange。
   - 枪口位置来自 `Player/VisualPivot/Muzzle`，方便在 Godot 编辑器里调。

6. 敌人生成器模板
   - `EnemySpawner` 围绕玩家半径随机补怪。
   - 默认目标数量约 120，保留导出参数可调整。
   - 每帧按 `refill_per_tick` 补齐，适合做视觉压力测试。

7. 伤害数字/受击反馈模板
   - `DamageText` 使用 `Label3D`、黑色描边、固定尺寸、向上漂浮淡出。
   - 命中生成彩色小火花。
   - 文字尺寸已经从首版巨型压屏改小，保留街机夸张感但不遮住主角。

8. UI 菜单模板
   - 已有 `DebugUI`：Health、Synergy、背包格、FPS、Debug 文本、敌人数量。
   - 当前是战斗 HUD 原型，不是完整菜单。
   - 后续可拆成 `HUD`、`PauseMenu`、`UpgradeMenu`、`SettingsMenu`。

9. 存档/设置模板
   - 今日尚未实装。
   - 已在 SOP 中预留 Godot Autoload + `user://settings.cfg` / `user://save.json` 的落地流程。

10. Steam 页面/宣传素材模板
   - 今日尚未实装。
   - 已在 SOP 中预留截图、GIF、短视频、Steam capsule、文案资产的开发流程。
