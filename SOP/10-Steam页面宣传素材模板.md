# Steam 页面/宣传素材模板 SOP

## 适用场景

用于把当前可玩切片转化为商店页和社媒宣传素材。今日尚未实装，本 SOP 是后续素材生产流程。

## 目标素材

1. Steam capsule 主图。
2. 5 张商店截图。
3. 1 个 6-10 秒循环 GIF。
4. 1 个 15-30 秒短 trailer。
5. 游戏一句话卖点。
6. Steam 短描述和长描述。

## 当前项目适合拍的卖点

- 仓鼠主角。
- 街霸训练场式 debug arena。
- 大量敌群。
- 彩色自动喷射弹幕。
- 夸张描边伤害数字。
- 经验球吸附。
- Juicy 合成音效。

## 截图 SOP

1. 固定分辨率：优先 1920x1080。
2. 使用训练场作为干净背景。
3. 保证画面里同时出现：
   - 主角。
   - 敌群。
   - 彩色弹幕。
   - 伤害数字。
   - UI。
   - 墙面 debug 信息。
4. 伤害字不要遮住主角脸。
5. 敌人数量控制在“丰富但可读”，推荐 80-140。

## GIF / Trailer SOP

1. 录制 10 秒战斗循环。
2. 前 2 秒展示敌群靠近。
3. 中间 5 秒展示彩色自动攻击和伤害字。
4. 最后 3 秒展示经验球吸附或升级 UI。
5. UI 保留一部分 Debug 风格，但不要让文字盖满画面。

## Steam 页面文案草稿

一句话：

> A tiny hamster survives a ridiculous debug arena with colorful auto-weapons, crunchy crits, and dangerously juicy cheese-powered upgrades.

中文定位：

> 一只仓鼠在街机测试场里用彩色自动武器暴打成群小怪，收集经验球，叠出越来越离谱的爽感构筑。

## 验收标准

- 第一张图 1 秒内能看懂：这是仓鼠 + Survivors + 爽感弹幕。
- 截图不是空场景，必须有战斗。
- 主角在截图中可识别。
- 伤害数字、弹幕、敌群三者同时出现但不互相糊住。
- 商店素材风格和游戏实际画面一致。

## 后续可做工具

- 新建 `PromoCapture` 场景。
- 固定相机和敌人生成脚本。
- 一键隐藏/显示 Debug UI。
- 一键设置高密度弹幕和固定伤害字。
